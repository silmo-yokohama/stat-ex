## 1. プロジェクト概要

### 1.1 コンセプト

**「5クリックかかる情報を1クリックで」**

横浜エクセレンス（B.LEAGUE B2）に特化した情報ダッシュボード。B.LEAGUE公式サイト・スポナビからスタッツデータを自動収集し、試合結果・選手スタッツ・チーム成績を一箇所に集約する。グラフを多用したスタイリッシュなダッシュボードUIで、既存サイトでは得られないワンストップの情報体験を提供する。

### 1.2 サイト情報

| 項目 | 値 |
|------|-----|
| サイト名 | **STAT-EX** |
| コンセプト | Stats × Excellence |
| ドメイン候補 | `exs.yokohama` / `ex-stats.yokohama` |
| ターゲット | 横浜エクセレンスの既存ファン、B2リーグに興味があるバスケファン |
| 公開範囲 | **初回リリースから一般公開** |

### 1.3 背景・課題

- スポナビでは TOP → BLEAGUE → B2 → 今日の試合 → エクセレンス と**5段階の導線**が必要
- B.LEAGUE公式も階層が深く、選手個人の前試合スタッツを見るには何度も戻る必要がある
- 既存スタッツサイトはリーグ全体を対象としており、特定チームに特化したダッシュボードが**存在しない**
- B2リーグのカバレッジは特に薄い

### 1.4 差別化ポイント

- 横浜エクセレンス**専門**のスタッツダッシュボード（競合ゼロ）
- リアルタイム試合更新（試合中に30秒自動更新）
- AI試合寸評（試合後に自動生成）
- YouTubeダイジェスト動画の自動埋め込み
- データビジュアライゼーション重視のUI

### 2.4 P3: 試合詳細ページ `/games/:scheduleKey`

1試合の全情報を集約するページ。**リアルタイム更新対応**。

#### 試合ステータスによる表示モード切替

| ステータス | 表示モード | 動作 |
|-----------|-----------|------|
| 試合前 | プレビューモード | 対戦相手情報、チケット購入リンク、試合開始カウントダウン |
| **試合中** | **ライブモード** | **30秒間隔の自動更新** + 手動更新ボタン。「LIVE」バッジ表示 |
| 試合後 | 結果モード | 確定データ + ダイジェスト動画 + AI寸評 |

#### リアルタイム更新の仕様

- `setInterval`（30秒）でB.LEAGUE `game_detail` を直接取得
- `visibilitychange` イベントで非表示タブ時は自動更新を停止
- 手動更新ボタン: 最終更新日時を表示し、ボタン押下で即時再取得
- リアルタイム取得データはDBに書き込まない（22:00バッチで確定データを格納）

#### 機能一覧

| ID | 機能 | 説明 | データソース |
|----|------|------|------------|
| P3-01 | スコアボードヘッダー | 大きなスコア + クォーター別スコア。LIVE/FINALバッジ | B.LEAGUE game_detail |
| P3-02 | Q別得点比較グラフ | Rechartsグループ棒グラフ。横浜EX(緑) vs 相手(グレー) | 同上 |
| P3-03 | チーム比較チャート | 水平バーチャート。FG%, 3P%, RB, AST, TO等を左右対称に表示 | 同上 |
| P3-04 | ボックススコア | タブ切替（横浜EX / 相手）。ソート可能テーブル。スターター=太字。列: MIN, PTS, FG%, 3P%, RB, AST, TO, ST, BS, F, EFF, +/- | 同上 |
| P3-05 | ダイジェスト動画 | YouTube iframe埋め込み。動画タイトル or プレイリストから試合を自動マッチング。該当動画がない場合は非表示 | YouTube RSS |
| P3-06 | AI試合寸評 | ボックススコア + クォーター別スコアから200〜300字の試合要約を自動生成 | AI API（Claude / OpenAI） |
| P3-07 | 試合情報 | 日付・会場・観客数・審判 | B.LEAGUE game_detail |
| P3-08 | チケット購入リンク | 試合前モードのみ表示 | 静的URL |

### 3.2 データアーキテクチャ

#### データフロー

```
[定期バッチ（毎日22:00）] ─── GitHub Actions
  ├→ B.LEAGUE公式: 試合結果・ボックススコア・選手スタッツ・順位表・チーム成績
  ├→ スポナビ: H2H対戦成績・インジュアリーリスト・チームリーダー
  ├→ Google News RSS / 公式HP RSS: ニュース記事
  ├→ YouTube RSS: 新着動画 → 試合とのマッチング
  └→ AI API: 新しい試合のボックススコアから寸評を生成
      ↓
  Supabase（PostgreSQL）に格納
      ↓
  Next.js フロントエンド → Supabaseから読み取って表示

[試合中のリアルタイム（例外フロー）]
  Next.js API Route
    └→ B.LEAGUE game_detail を30秒ごとに直接取得
        └→ クライアントに返却（DBには書き込まない）
```

#### データ更新タイミング

| データ | 更新タイミング | 更新元 |
|--------|-------------|--------|
| 試合結果・ボックススコア | 毎日22:00バッチ | B.LEAGUE game_detail |
| 選手スタッツ（シーズン平均・通算） | 毎日22:00バッチ | B.LEAGUE roster_detail |
| 選手の試合別成績ログ | 毎日22:00バッチ | ボックススコアから各選手分を抽出 |
| チーム成績・順位表 | 毎日22:00バッチ | B.LEAGUE standings + club_detail |
| H2H対戦成績・インジュアリー | 毎日22:00バッチ | スポナビ widget |
| ニュース | 毎日22:00バッチ | RSS |
| YouTube動画 | 毎日22:00バッチ | YouTube RSS |
| AI寸評 | 毎日22:00バッチ（新試合のみ） | AI API（生成後DBに保存。再生成しない） |
| **試合中ライブスコア** | **30秒ごとにオンデマンド** | **B.LEAGUE game_detail を直接取得** |

### 3.3 データ蓄積方針

| 方針 | 説明 |
|------|------|
| 蓄積開始時期 | 2025-26シーズン（B2昇格シーズン）以降 |
| 過去データ | 2020-21〜2024-25はB3所属のため、B.LEAGUE公式・スポナビともにデータなし |
| **移籍選手の扱い** | **移籍した選手のデータは永続的に保持する**。在籍期間を記録し、横浜EX在籍時のスタッツのみを表示。他チームでの成績は追わない |
| シーズン跨ぎ | 来シーズン以降も継続蓄積。シーズンごとにデータを区分管理する |

### 3.4 データベース（Supabase）

#### 主要テーブル設計（概要）

| テーブル | 説明 | 主要カラム |
|---------|------|-----------|
| `seasons` | シーズンマスタ | id, year, name, start_date, end_date |
| `games` | 試合データ | id, schedule_key, season_id, date, opponent, home_away, score_home, score_away, status, venue, attendance |
| `box_scores` | ボックススコア | id, game_id, player_id, team_side, minutes, pts, fgm, fga, fg_pct, ... , eff, plus_minus |
| `players` | 選手マスタ | id, bleague_player_id, name, number, position, height, weight, birthdate, image_url |
| `player_seasons` | 選手のシーズン在籍情報 | id, player_id, season_id, is_active, joined_date, left_date |
| `team_stats` | チーム成績（シーズン単位） | id, season_id, wins, losses, ... |
| `standings` | B2順位表 | id, season_id, team_id, rank, wins, losses, win_pct, ... |
| `news` | ニュース記事 | id, source, title, url, published_at, thumbnail_url |
| `videos` | YouTube動画 | id, video_id, title, published_at, thumbnail_url, game_id(nullable) |
| `game_comments` | AI試合寸評 | id, game_id, content, generated_at |
| `h2h_records` | 対戦成績 | id, season_id, opponent_team_id, wins, losses, avg_points_for, avg_points_against |

## 7. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| B.LEAGUE公式サイトの構造変更 | 高 | スクレイピング部分をモジュール化し、変更時に最小限の修正で対応。GitHub Actionsのエラー通知（LINE）で即座に検知 |
| 利用規約への抵触 | 中 | 非商用・個人利用を明示。適切なリクエスト頻度（1秒以上の間隔）。データ出典を必ず記載。フッターに出典表示 |
| スポナビ Widget URLの変更 | 低 | サブソースなのでメインに影響なし。壊れた場合はB.LEAGUE公式のみで運用可能な設計にする |
| Supabase無料枠の制限 | 低 | B2の試合データは年間60試合程度。データ量は小さく、無料枠で十分 |
| AI API利用料の増加 | 低 | 1試合あたり数円。月30試合想定で月額100〜300円。予算内 |
| シーズンオフの訪問減少 | 中 | ニュースの継続配信 + 蓄積データの閲覧で最低限のコンテンツを維持。来シーズン開幕時にリピーターを獲得する |
